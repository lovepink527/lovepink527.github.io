git是远程仓库与本地仓库链接的桥梁
开发过程中有开发分支与线上分支
如何进行版本控制呢

  • 平时开发常用git命令

  • 基本上平时这几个命令就够使了
  • 但是如果出现远程仓库多个分支、本地仓库多个分支如何解决呢？？？？
  • 默认本地仓库和远程仓库都会有一个master分支并且是关联的不用配置
  • 如果我们又想新建一个dev分支，这时我们需要做其他工作了
  • 这样子远程仓库dev 与本地仓库dev就链接好了
  • 在本地dev分支git pull git push 默认分支都是远程仓库的dev
